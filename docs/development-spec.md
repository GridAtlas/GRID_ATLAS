# GRID ATLAS 開発仕様書

## 1. コンセプト

GRID ATLASは、建物名、地名、地形、道路、外部地図タイルを一切表示しない地図アプリである。画面の中心は抽象的な格子で、ユーザーが登録した地球上の地点だけをオレンジ色の点として表示する。

通常の地図が「周囲の情報を読む」道具であるのに対し、GRID ATLASは「自分が記録した地点同士の位置関係を読む」道具として設計する。

## 2. 設計原則

- 地図上には登録地点、地点間の線、計測補助だけを表示する。
- 建物名、施設名、地名、道路、地形、行政境界、外部POIは表示しない。
- 地点名やコメントは、点を選択した詳細パネルでのみ表示する。
- スマホは現地での地点登録、写真、コメント入力を主目的にする。
- PCは登録地点同士の位置関係、線、距離、分析を主目的にする。
- 実位置は緯度経度で保持し、画面表現は抽象格子に寄せる。
- 画面上に「原点」「0m地点」のようなローカル座標の意味付けを出さない。

## 3. 想定ユーザー体験

### スマホ

- 現地で現在地を取得する。
- 一般的な地図アプリから共有された座標付きURLやテキストを読み取る。
- 写真を撮る、または選択する。
- コメントを残す。
- 登録後、地点は地球上の実位置を持つ点として格子に反映される。

### PC

- 登録済み地点を広い画面で俯瞰する。
- 点同士を線で結ぶ。
- 2点間の地球上の直線距離を測る。
- 複数地点を選び、任意の地点をスタートにした巡回順を確認する。
- 登録地点、リンク距離、合計距離、最長距離を確認する。
- JSONでデータを入出力する。

## 5. 座標モデル

GRID ATLASは全地点を緯度経度で保持する。画面描画時だけWeb Mercatorに投影し、抽象格子上の点として表示する。

### 内部保持

- `geo.lat`: 緯度
- `geo.lng`: 経度
- `geo.accuracy`: 位置取得精度。取得できた場合のみ保存する。
- `x`: Web Mercator投影後のX座標。描画用。
- `y`: Web Mercator投影後のY座標。描画用。

### 距離

距離分析では、描画用の平面距離ではなく緯度経度から球面距離を計算する。MVPではHaversine式を使う。

巡回順計算では、選択した複数地点を対象に、指定したスタート地点から全地点を1回ずつ訪問する順番を提示する。道路網や通行可能性は考慮せず、地点間の直線距離だけを最小化対象にする。12点までは厳密計算、13点以上は近似計算とする。

外部地図アプリからの共有は、PWAのShare Targetと共有テキスト貼り付けの2系統で受ける。緯度経度がURLや本文に含まれる場合だけ登録フォームへ反映する。Appleマップの「北35.75729°, 東139.85496°」のような方位付き表記にも対応する。短縮URLだけの場合は、ブラウザのCORS制約によりクライアント単体では展開しない。

### 表示

- 格子線に地名や道路名は載せない。
- 原点線や0m地点の表示はしない。
- 緯度経度は登録フォームと詳細パネルでのみ扱う。
- 外部地図を開く場合は、選択地点の緯度経度だけを外部サービスへ渡す。

## 6. データモデル

### Workspace

```json
{
  "version": 2,
  "projection": "web-mercator",
  "points": [],
  "links": []
}
```

### Point

```json
{
  "id": "uuid",
  "x": 15550408.9,
  "y": 4257980.7,
  "title": "詳細パネル用の見出し",
  "note": "コメント",
  "photo": "data-url-or-storage-url",
  "photoName": "photo.jpg",
  "geo": {
    "lat": 35.681236,
    "lng": 139.767125,
    "accuracy": 18
  },
  "createdAt": "2026-07-07T00:00:00.000Z"
}
```

### Link

```json
{
  "id": "uuid",
  "a": "point-id",
  "b": "point-id",
  "createdAt": "2026-07-07T00:00:00.000Z"
}
```

## 7. UI要件

### 共通

- 画面の主役は格子キャンバスにする。
- 点はオレンジ色の丸で表示する。
- 点の横に名称を出さない。
- 選択中の点はリングで強調する。
- リンクは点同士を結ぶ線で表示する。
- 巡回モードでは複数地点を選択し、選択順または計算結果の番号を点の近くに表示する。
- ホイールまたはボタンで拡大縮小できる。
- ドラッグでパンできる。
- 地図由来の地名、道路、地形、建物情報は表示しない。

### スマホ

- 登録フォームを操作しやすい位置に置く。
- 写真入力はカメラ起動を許可する。
- 現在地取得ボタンで緯度経度を埋める。
- 共有された座標付きURLやテキストを地点登録フォームに反映できる。
- 片手操作でも登録できる程度にボタンを大きくする。

### PC

- 格子キャンバスを広く取る。
- 右側に地点詳細、距離分析、データ操作をまとめる。
- リンク距離一覧、合計距離、最長距離を見られるようにする。
- 巡回ルート一覧では、訪問順、前地点からの距離、累積距離、合計距離を見られるようにする。

## 8. 技術構成

### 初期プロトタイプ

- HTML
- CSS
- JavaScript
- Canvas 2D
- LocalStorage
- Node.js標準ライブラリによる開発サーバー

依存パッケージを持たないため、ネットワークなしで起動できる。

### 本番候補

- フロントエンド: TypeScript + ReactまたはSvelteKit
- API: Node.jsまたはCloudflare Workers
- DB: PostgreSQL + PostGIS
- 写真保存: S3互換オブジェクトストレージ
- 認証: Passkeyまたはメールリンク
- 同期: ワークスペース単位の差分同期
- PWA: オフライン登録、復帰時同期

外部地図プロバイダーは使わない。

## 9. セキュリティとプライバシー

- 位置情報はユーザー操作時のみ取得する。
- 地点名、コメント、写真はワークスペース権限で保護する。
- 共有URLは推測困難なIDにする。
- 写真アップロード時はEXIF削除を検討する。
- サーバー同期ではユーザーごとの保存容量を制限する。

- 原点や0m地点の表示がない。
