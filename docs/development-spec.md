# GRID ATLAS 開発仕様書

## 1. コンセプト

GRID ATLASは、建物名、地名、地形、道路、外部地図タイルを一切表示しない地図アプリである。画面の中心は抽象的な格子で、ユーザーが登録した地点だけをオレンジ色の点として表示する。

通常の地図が「周囲の情報を読む」道具であるのに対し、GRID ATLASは「自分が記録した地点同士の位置関係を読む」道具として設計する。

## 2. 設計原則

- 地図上には登録地点、地点間の線、計測補助だけを表示する。
- 建物名、施設名、地名、道路、地形、行政境界、外部POIは表示しない。
- 地点名やコメントは、点を選択した詳細パネルでのみ表示する。
- スマホは現地での地点登録、写真、コメント入力を主目的にする。
- PCは登録地点同士の位置関係、線、距離、分析を主目的にする。
- 実距離は保持するが、画面表現は抽象格子に寄せる。

## 3. 想定ユーザー体験

### スマホ

- 現地で地点を登録する。
- 現在地から格子座標を生成する。
- 写真を撮る、または選択する。
- コメントを残す。
- 登録後、点として格子に反映される。

### PC

- 登録済み地点を広い画面で俯瞰する。
- 点同士を線で結ぶ。
- 2点間の直線距離を測る。
- 登録地点、リンク距離、合計距離、最長距離を確認する。
- JSONでデータを入出力する。

## 4. MVP範囲

### 実装する

- レスポンシブWebアプリ
- 格子キャンバス
- 地点登録
- 写真、コメント保存
- 点クリックによる詳細表示
- 地点間リンク
- 2点間の直線距離計測
- ローカル保存
- JSONエクスポート、インポート

### MVPでは実装しない

- 外部地図タイル表示
- 地名、建物名、道路名の表示
- ルート検索
- ナビゲーション
- 複数ユーザー共有
- サーバー同期
- 高度なGIS解析

## 5. 座標モデル

GRID ATLASは画面上に緯度経度を直接表示しない。内部的には、地点をローカル平面座標として扱う。

### 基本単位

- `x`: 原点から東西方向のメートル
- `y`: 原点から南北方向のメートル
- 距離: `sqrt((x2 - x1)^2 + (y2 - y1)^2)`

### 現在地からの変換

スマホで現在地を取得した場合、最初に取得した地点をワークスペースの原点にする。以後の地点は、原点からの相対メートル座標に変換する。

近距離利用を想定し、初期版では簡易的な正距円筒近似を使う。

- `y = (lat - originLat) * 111320`
- `x = (lng - originLng) * 111320 * cos(originLat)`

将来、広域利用が必要になった場合は、ワークスペースごとに投影法を持たせる。

## 6. データモデル

### Workspace

```json
{
  "version": 1,
  "origin": {
    "lat": 35.0,
    "lng": 139.0,
    "createdAt": "2026-07-07T00:00:00.000Z"
  },
  "points": [],
  "links": []
}
```

### Point

```json
{
  "id": "uuid",
  "x": 12.4,
  "y": -8.1,
  "title": "詳細パネル用の見出し",
  "note": "コメント",
  "photo": "data-url-or-storage-url",
  "photoName": "photo.jpg",
  "geo": {
    "lat": 35.0,
    "lng": 139.0,
    "accuracy": 18
  },
  "createdAt": "2026-07-07T00:00:00.000Z"
}
```

### Link

```json
{
  "id": "uuid",
  "a": "point-id",
  "b": "point-id",
  "createdAt": "2026-07-07T00:00:00.000Z"
}
```

## 7. UI要件

### 共通

- 画面の主役は格子キャンバスにする。
- 点はオレンジ色の丸で表示する。
- 点の横に名称を出さない。
- 選択中の点はリングで強調する。
- リンクは点同士を結ぶ線で表示する。
- ホイールまたはボタンで拡大縮小できる。
- ドラッグでパンできる。

### スマホ

- 登録フォームを操作しやすい位置に置く。
- 写真入力はカメラ起動を許可する。
- 現在地取得ボタンで座標を埋める。
- 片手操作でも登録できる程度にボタンを大きくする。

### PC

- 格子キャンバスを広く取る。
- 右側に地点詳細、距離分析、データ操作をまとめる。
- リンク距離一覧、合計距離、最長距離を見られるようにする。

## 8. 技術構成

### 初期プロトタイプ

- HTML
- CSS
- JavaScript
- Canvas 2D
- LocalStorage
- Node.js標準ライブラリによる開発サーバー

依存パッケージを持たないため、ネットワークなしで起動できる。

### 本番候補

- フロントエンド: TypeScript + ReactまたはSvelteKit
- API: Node.jsまたはCloudflare Workers
- DB: PostgreSQL + PostGIS
- 写真保存: S3互換オブジェクトストレージ
- 認証: Passkeyまたはメールリンク
- 同期: ワークスペース単位の差分同期
- PWA: オフライン登録、復帰時同期

外部地図プロバイダーは使わない。

## 9. セキュリティとプライバシー

- 位置情報はユーザー操作時のみ取得する。
- 地点名、コメント、写真はワークスペース権限で保護する。
- 共有URLは推測困難なIDにする。
- 写真アップロード時はEXIF削除を検討する。
- サーバー同期ではユーザーごとの保存容量を制限する。

## 11. 受け入れ条件

- 地図タイルや地名が表示されない。
- 登録した地点だけが格子上に表示される。
- 点をクリックすると写真とコメントを確認できる。
- 2点間に線を引ける。
- 2点間の直線距離を確認できる。
- スマホ幅でも地点登録ができる。
- PC幅では分析パネルが視認しやすい。
