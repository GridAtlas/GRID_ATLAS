# GRID ATLAS 入出力機能 実装指示書

作成日: 2026-07-12
対象: 地点リスト入出力、観察記録入出力

## 目的

既存の「データ > 書き出し/読み込み」はワークスペース全体のバックアップとして残す。
追加機能として、用途を分けた2種類のファイル入出力を実装する。

- 地点リスト: 登録地点だけを交換、追加、置き換えできる。
- 観察記録: 起点、対象、実軌道、距離指標をあとから読み込んで再表示できる。

## 既存機能との関係

既存の `exportWorkspace()` / `importWorkspaceFile()` は変更しすぎない。
新機能は原則として別関数・別ボタンで追加する。

推奨UI:

- データパネル内に小見出しを追加する。
- 「全体」: 既存の `書き出し / 読み込み / 初期化`
- 「地点リスト」: `地点を書き出し / 地点を読み込み`
- 「観察記録」: `観察を書き出し / 観察を読み込み`

## 地点リストファイル

### ファイル形式

JSON形式。トップレベルに `type` と `version` を持たせる。

```json
{
  "type": "grid-atlas-points",
  "version": 1,
  "exportedAt": "2026-07-12T00:00:00.000Z",
  "points": [
    {
      "id": "optional-original-id",
      "title": "地点名",
      "note": "コメント",
      "photo": "data:image/...",
      "photoName": "photo.jpg",
      "geo": {
        "lat": 35.681236,
        "lng": 139.767125,
        "accuracy": 12.3
      },
      "createdAt": "2026-07-12T00:00:00.000Z"
    }
  ]
}
```

### 書き出し

対象は `state.points` のみ。
現在地、線、巡回結果、対象、起点、観察記録は含めない。

ファイル名:

```text
grid-atlas-points-YYYY-MM-DD.json
```

### 読み込み

読み込み時に以下を選択させる。

- 置き換え: 現在の登録地点をすべて消して、読み込んだ地点だけにする。
- 追加: 現在の登録地点に読み込んだ地点を追加する。
- キャンセル

最初は `window.confirm()` の2段階でもよい。
将来的には専用モーダルにする。

推奨フロー:

1. JSONを読む。
2. `type === "grid-atlas-points"` と `Array.isArray(points)` を検証する。
3. 各地点を `normalizePoint(point, null)` で正規化する。
4. 置き換えの場合:
   - `state.points = importedPoints`
   - `state.links = []`
   - `state.selection = []`
   - `state.routeSelectionIds = []`
   - `clearRouteStartState()` 相当で起点を解除
   - `clearTarget({ render: false })`
   - `state.routeResult = null`
5. 追加の場合:
   - 既存IDと衝突する読み込み地点には `createId()` で新IDを付ける。
   - 写真、コメント、作成日時は保持する。
   - 線や巡回状態は変更しない。
6. `persistWorkspace()` する。
7. `fitToPoints()` で表示を合わせる。

### 受け入れ条件

- 登録地点だけを書き出せる。
- 置き換え読み込みで既存の線や対象が残らない。
- 追加読み込みで既存地点が消えない。
- ID衝突があっても地点が欠落しない。
- 壊れたJSONでは `読み込みエラー` を表示し、既存データを変更しない。

## 観察記録ファイル

### ファイル形式

観察記録はワークスペースとは独立したファイルにする。

```json
{
  "type": "grid-atlas-observation",
  "version": 1,
  "exportedAt": "2026-07-12T00:00:00.000Z",
  "startedAt": "2026-07-12T00:00:00.000Z",
  "endedAt": "2026-07-12T00:30:00.000Z",
  "start": {
    "id": "start-id-or-current",
    "title": "起点",
    "geo": { "lat": 35.0, "lng": 139.0 },
    "x": 0,
    "y": 0
  },
  "target": {
    "id": "target-id",
    "title": "対象",
    "geo": { "lat": 35.1, "lng": 139.1 },
    "x": 0,
    "y": 0
  },
  "trail": [
    {
      "id": "__current_location__",
      "title": "現在地",
      "geo": { "lat": 35.01, "lng": 139.01, "accuracy": 10 },
      "x": 0,
      "y": 0,
      "recordedAt": "2026-07-12T00:01:00.000Z"
    }
  ],
  "metrics": {
    "remaining": 1234.5,
    "traveled": 2345.6,
    "ratio": 1.23
  }
}
```

### 記録終了時の保存確認

観察モード解除時に、実軌道が1点以上ある場合は保存確認を出す。

推奨メッセージ:

```text
観察記録を保存しますか。
```

処理順:

1. 観察解除確認を出す。
2. 解除が承認されたら、実軌道がある場合に保存確認を出す。
3. 保存する場合はJSONをダウンロードする。
4. その後、追従停止・対象解除・起点解除など本来の操作を続行する。

注意:

- GPSエラーなど自動停止時は保存確認を出しすぎない。まずは手動操作時のみでよい。
- 保存前に `observationMetrics()` を呼び、距離指標をスナップショットとして持つ。

ファイル名:

```text
grid-atlas-observation-YYYY-MM-DD-HHmm.json
```

### 読み込み

読み込んだ観察記録は、通常の登録地点や現在の追従状態とは別の「読み込み観察レイヤー」として表示する。

追加する状態案:

```js
state.loadedObservation = null;
```

描画方針:

- `state.observationTrail` がある時は現在の観察を優先表示。
- それがない時に `state.loadedObservation` を表示。
- 読み込み観察は編集対象にしない。
- 上部情報欄は `読込観察 起点 → 対象 | 実 ... | 道直比 ...` のように表示する。

読み込み時の選択肢:

- 表示する: 現在の観察レイヤーに読み込み表示する。
- キャンセル

現在観察中の場合:

- 上書き表示前に確認を出す。
- 観察中の実軌道があるなら、先に保存確認を出す。

### 受け入れ条件

- 観察終了時に記録保存を選べる。
- 保存した観察JSONを読み込んで、起点、対象、実軌道が再表示される。
- 読み込み観察は登録地点や線を汚さない。
- 現在の観察中データを誤って消す操作では確認が出る。
- 壊れたJSONでは `読み込みエラー` を表示し、現在状態を変更しない。

## 実装順序

1. 地点リスト書き出しを実装する。
2. 地点リスト読み込みの追加/置き換えを実装する。
3. 観察記録のJSON生成関数を実装する。
4. 観察終了時の保存確認を実装する。
5. 観察記録読み込み用の `state.loadedObservation` と描画を実装する。
6. 上部情報欄に読み込み観察の要約を表示する。
7. 手動テスト後、PWAキャッシュバージョンを更新する。

## 注意点

- 既存の全体バックアップ形式は壊さない。
- 既存の `docs/2026-07-10_claude_proposal.md` はこの作業では参照・変更しない。
- 観察記録はローカルファイルとして扱い、外部送信しない。
- 将来のアプリ化を考え、ファイル形式の `type` と `version` は必ず保持する。
